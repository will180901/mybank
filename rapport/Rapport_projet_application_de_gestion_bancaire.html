1. Introduction

MyBank est une application bancaire complète développée avec Qt/C++ et SQLite. Elle offre une solution moderne pour la gestion des comptes bancaires, des transactions et des opérations financières.

Objectifs

  • Fournir une gestion complète des comptes bancaires
  • Permettre des opérations sécurisées (dépôts, retraits, virements)
  • Offrir une visualisation claire de l'historique
  • Assurer une expérience utilisateur moderne
  • Garantir la sécurité des données financières

Architecture

  • Frontend: Qt Widgets avec animations
  • Backend: Logique métier en C++
  • Base de données: SQLite
  • Sécurité: Chiffrement des données sensibles

Technologies

  • C++17
  • Qt 6 Framework
  • SQLite3
  • QWidgets pour l'interface
  • CSS pour le style

Modèle Conceptuel de Données (MCD)

// Entité Utilisateur
Table Utilisateur {
  id int [pk]
  nom_complet varchar
  email varchar
  mot_de_passe varchar
}

Table Compte {
  id int [pk]
  numero_compte varchar
  type_compte varchar
  solde decimal
  id_utilisateur int
}

Ref: Compte.id_utilisateur > Utilisateur.id

Table Transaction {
  id int [pk]
  type_operation varchar
  montant decimal
  id_compte_source int
  id_compte_destination int [null]
  date_operation datetime
}

Ref: Transaction.id_compte_source > Compte.id
Ref: Transaction.id_compte_destination > Compte.id

Modèle Physique de Données (MPD)

Table Colonne Type Contraintes
Utilisateur id INTEGER PRIMARY KEY AUTOINCREMENT
nom_complet VARCHAR(100) NOT NULL
email VARCHAR(100) UNIQUE, NOT NULL
mot_de_passe VARCHAR(255) NOT NULL
Compte id INTEGER PRIMARY KEY AUTOINCREMENT
numero_compte VARCHAR(20) UNIQUE, NOT NULL
type_compte VARCHAR(10) 'courant' ou 'epargne'
solde DECIMAL(15,2) DEFAULT 0.00
id_utilisateur INTEGER FOREIGN KEY REFERENCES Utilisateur(id)
date_creation DATETIME NOT NULL

4. Difficultés Rencontrées

Gestion des transactions

Assurer l'atomicité des opérations bancaires était un défi technique majeur.

  • Solution: Utilisation des transactions SQL
  • Rollback en cas d'erreur
  • Verrouillage des ressources

Synchronisation UI/BD

Maintenir l'interface à jour avec la base de données en temps réel.

  • Solution: Système de signaux/slots Qt
  • Événements de mise à jour
  • Cache local pour performances

Sécurité

Protection des données sensibles et prévention des fraudes.

  • Chiffrement des mots de passe
  • Validation des entrées utilisateur
  • Contrôle d'accès strict

Solutions techniques

Problème Solution Résultat
Performances historique Pagination et indexation Temps de chargement divisé par 5
UI complexe Custom Widgets Qt Expérience utilisateur fluide
Gestion thèmes Système de gestion centralisé Changement de thème en temps réel
Gestion erreurs Exceptions et logs détaillés Débogage facilité
